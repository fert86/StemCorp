Solución para Error "Page not found" en Netlify

Aprende cómo resolver el error 404 cuando despliegas tu sitio en Netlify. Soluciones prácticas y efectivas para el enrutamiento en aplicaciones de una sola página (SPA).

Ver Solución

¿Por qué ocurre el error "Page not found"?

Este error generalmente ocurre cuando Netlify no puede encontrar la ruta solicitada porque estás usando enrutamiento del lado del cliente (como React Router) y Netlify busca archivos físicos que no existen.

Solución: Archivo _redirects

Crea un archivo llamado _redirects en tu carpeta pública con el siguiente contenido:

# Configuración para Netlify - Single Page Application /* /index.html 200

Este archivo le indica a Netlify que redirija todas las rutas a tu archivo index.html, permitiendo que tu aplicación de una sola página maneje el enrutamiento.

Pasos para resolver el problema

Paso 1: Crear archivo _redirects

Crea un archivo llamado _redirects (sin extensión) en tu carpeta pública (public/ para React).

Paso 2: Configurar contenido

Añade esta línea al archivo: /* /index.html 200. Esto redirige todas las rutas a index.html.

Paso 3: Configuración netlify.toml

Crea un archivo netlify.toml en la raíz de tu proyecto con la configuración de despliegue.

Paso 4: Redesplegar aplicación

Realiza un nuevo deploy en Netlify después de hacer estos cambios para que surtan efecto.

Configuración completa para netlify.toml

Para una configuración más avanzada, crea un archivo netlify.toml en la raíz de tu proyecto:

[build] publish = "build" command = "npm run build" [[redirects]] from = "/*" to = "/index.html" status = 200 # Para manejar errores 404 [[redirects]] from = "/*" to = "/index.html" status = 404
